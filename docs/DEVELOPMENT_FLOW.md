## 阶段一：需求分析

**目标**：搞清楚要做什么，输出需求文档。

### 步骤

1. **列出功能清单**
   - 核心功能（必须有的）
   - 扩展功能（以后再加的）
   - 每个功能的具体描述

2. **绘制用户流程**
   - 用户打开页面看到什么？
   - 点击按钮后发生什么？
   - 操作路径是什么？

3. **确定页面结构**
   - 首页 / 游戏页 / 结算页 ...
   - 有哪些弹窗 / 交互组件

4. **输出**：一份清晰的功能列表，越详细越好

### 示例

```
项目：百万富翁答题游戏

核心功能：
  - 用户点击「开始游戏」进入答题
  - 每题 4 个选项，选择后显示对错
  - 15 道题，难度递增
  - 奖金梯显示当前奖金
  - 3 条救命热线（50:50 / 问观众 / 打电话）
  - 可中途提现走人

扩展功能：
  - 排行榜
  - 音效开关
  - 暗色/亮色主题
```

## 阶段二：技术选型

**目标**：根据需求选择合适的技术方案。

### 方案对比

| 方案 | 适用场景 | 部署 | 复杂度 |
|------|---------|------|--------|
| **纯前端**（HTML + CSS + JS） | 工具类、游戏、展示页 | Netlify / GitHub Pages | ★☆☆ |
| **前端框架**（React / Vue） | 复杂交互、大型应用 | Netlify / Vercel | ★★☆ |
| **前后端分离**（前端 + Node.js） | 需要数据库、用户系统 | Netlify + 后端服务 | ★★★ |

### 我们采用的方案

```
技术栈：
  - 前端：HTML + CSS + JavaScript（纯前端，无需构建工具）
  - 音效：Web Audio API（无需外部音频文件）
  - 数据存储：LocalStorage（本地存储）
  - 部署：GitHub + Netlify
```

**优点**：
- 不需要安装 Node.js / npm
- 不需要构建打包
- 一个 HTML 就能跑
- 部署最简单，Netlify 直接托管

## 阶段三：项目搭建

**目标**：创建标准化的项目目录结构。

### 目录结构规范

```
project-name/
├── index.html              # 主页面（入口）
├── assets/
│   ├── css/
│   │   └── style.css       # 样式文件
│   ├── js/
│   │   ├── app.js          # 主逻辑（初始化、路由）
│   │   ├── game.js         # 游戏核心逻辑
│   │   └── data.js         # 数据（题目、配置等）
│   ├── images/             # 图片资源
│   └── sounds/             # 音效资源
├── README.md               # 项目说明
└── DEPLOY.md               # 部署说明
```

### 初始化命令

```bash
# 克隆远程仓库（如果是从 GitHub 开始）
git clone https://github.com/你的用户名/项目名.git

# 或本地初始化后再关联远程
mkdir 项目名
cd 项目名
git init
git remote add origin https://github.com/你的用户名/项目名.git
```

## 阶段四：编码实现

**目标**：按模块开发，每个模块完成后提交一次 Git。

### 开发顺序

1. **HTML 结构** → 先搭骨架
2. **CSS 样式** → 再美化外观
3. **JavaScript 逻辑** → 最后实现交互

### 编码原则

- 每完成一个功能就 `commit` 一次
- 每次 `commit` 后 `push` 到 GitHub（Netlify 会自动部署）
- 先做核心功能，再做锦上添花

### Git 提交流程

```bash
# 1. 查看当前状态
git status

# 2. 添加文件
git add .

# 3. 提交
git commit -m "feat: 完成题目展示功能"

# 4. 推送到 GitHub
git push origin main
```

> **每次 `git push` 后，Netlify 会自动重新部署，链接不变，内容更新。**

## 阶段五：本地测试

**目标**：在本地验证功能正常。

### 测试方法

1. **直接打开 HTML**
   - 在浏览器中双击 `index.html` 即可运行
   - 纯前端项目本地打开即可测试全部功能

2. **测试清单**
   - [ ] 所有按钮点击有反馈
   - [ ] 页面在不同屏幕尺寸下正常（手机/平板/电脑）
   - [ ] 功能流程完整走通
   - [ ] 没有控制台报错

3. **Bug 修复流程**
   ```
   发现 Bug → 修改代码 → 本地验证 → git commit → git push → 线上同步修复
   ```

## 阶段六：部署上线

**目标**：将项目部署到公网，生成可分享的链接。

### 方式一：Netlify（推荐）

1. 打开 https://app.netlify.com/
2. 点击 **"Add new site"** → **"Import an existing project"**
3. 选择 **GitHub**
4. 找到对应的仓库
5. 部署设置：
   - **Branch to deploy**：`main`
   - **Build command**：留空（纯前端不需要）
   - **Publish directory**：留空（默认根目录）
6. 点击 **"Deploy"**
7. 等待 1-2 分钟，生成链接 `https://xxx.netlify.app`

### 方式二：GitHub Pages

1. 进入 GitHub 仓库 → **Settings** → **Pages**
2. Source 选 **"Deploy from a branch"**
3. Branch 选 **main**，目录选 **/(root)**
4. 点 **Save**
5. 生成链接 `https://你的用户名.github.io/仓库名`

## 阶段七：线上验证

**目标**：确认线上链接正常工作。

### 验证清单

- [ ] 打开链接，页面正常加载
- [ ] 所有功能在线上正常
- [ ] 手机端访问正常
- [ ] 分享给朋友测试访问

### 如果出现问题

| 问题 | 解决方法 |
|------|---------|
| 页面空白 | 检查文件路径是否大小写正确 |
| 样式不对 | CSS 文件路径是否正确？ |
| 功能异常 | 打开浏览器控制台（F12）看报错 |
| 链接打不开 | 等 1-2 分钟再试，部署可能需要时间 |

## 阶段八：迭代更新

**目标**：持续改进项目。

### 更新流程

```bash
# 1. 修改代码

# 2. 提交推送
git add .
git commit -m "feat: 增加排行榜功能"
git push origin main

# 3. Netlify 自动重新部署（约 1-2 分钟）
# 4. 同一链接自动更新，用户无感知
```

> **核心优势**：链接不变，内容自动更新。发出去的链接永久有效。

### 版本记录

在 `index.html` 或 `README.md` 中维护更新日志：

```markdown
## 更新日志

### v1.1.0 (2026-07-06)
- 新增排行榜功能
- 修复题目切换闪烁问题
- 优化手机端显示

### v1.0.0 (2026-07-05)
- 初始版本上线
- 核心答题功能
- 3 条救命热线
```

## Git 提交规范

统一使用以下前缀，方便追踪：

| 前缀 | 含义 | 示例 |
|------|------|------|
| `feat:` | 新功能 | `feat: 完成答题逻辑` |
| `fix:` | Bug 修复 | `fix: 修复答案判断错误` |
| `style:` | 样式调整 | `style: 优化首页布局` |
| `refactor:` | 代码重构 | `refactor: 重构游戏状态管理` |
| `docs:` | 文档更新 | `docs: 更新 README` |
| `perf:` | 性能优化 | `perf: 优化题目加载速度` |

## 常见问题

### Q1：改代码后怎么更新线上链接？
直接 `git push` 即可，Netlify 自动重新部署，链接不变。

### Q2：国内用户能访问 Netlify 链接吗？
大部分地区可以正常访问，速度还可以。如果追求更快，可以换用腾讯云静态托管或阿里云 OSS。

### Q3：纯前端项目能做用户登录吗？
纯前端做不了真正的用户登录（没有后端和数据库）。但可以用 LocalStorage 做简单的本地账号模拟。

### Q4：需要域名吗？
不需要。Netlify 生成的 `xxx.netlify.app` 链接可以直接使用。如果有自己的域名，也可以绑定。

### Q5：流量大了会收费吗？
Netlify 免费版每月 100GB 带宽，个人项目完全够用。
