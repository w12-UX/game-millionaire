# 《一掷千金 Deal or No Deal》前端 Demo 产品需求文档

## 1. 项目概述

| 项目 | 内容 |
|------|------|
| **项目名称** | 一掷千金小游戏 Demo（Deal or No Deal） |
| **项目定位** | 复刻美国经典综艺《Deal or No Deal》核心玩法的轻量化前端小游戏 |
| **核心体验** | 概率博弈 + 心理决策，无答题、无闯关，纯开箱淘汰 + 银行家报价抉择 |
| **技术形态** | 纯前端 HTML+CSS+JS，零服务端依赖 |
| **部署方式** | 本地可运行 → GitHub → Netlify 一键部署 |

## 2. 游戏核心规则

### 2.1 基础流程

1. **选专属箱子**：玩家从 26 个箱子中选择 1 个作为专属箱子，全程不提前打开
2. **多轮开箱淘汰**：分多轮依次打开剩余箱子，开出的金额直接淘汰出局，实时更新场上剩余奖金池
3. **银行家报价**：每轮开箱结束后，系统自动给出收购报价，玩家二选一：
   - **Deal**（成交）：接受奖金，游戏结束
   - **No Deal**（不成交）：拒绝报价，继续下一轮
4. **终极抉择**：最后剩 2 箱时，玩家可选择交换/不交换，最终开箱结算

### 2.2 金额池（26 档固定）

```
$0.01, $1, $5, $10, $25, $50, $75, $100,
$200, $300, $400, $500, $750, $1,000,
$5,000, $10,000, $25,000, $50,000, $75,000,
$100,000, $200,000, $300,000, $400,000,
$500,000, $750,000, $1,000,000
```

### 2.3 开箱轮次规则

| 轮次 | 本轮开箱数 |
|------|-----------|
| 第 1 轮 | 6 个 |
| 第 2 轮 | 5 个 |
| 第 3 轮 | 4 个 |
| 第 4 轮 | 3 个 |
| 第 5 轮 | 2 个 |
| 第 6 轮起 | 每轮 1 个 |
| 结束条件 | 剩余 2 箱触发终极抉择 |

### 2.4 银行家报价算法

**核心逻辑**：基于场上剩余所有金额的平均值，结合游戏进度动态浮动，模拟原版综艺前期保守压价、后期激进抬价的效果。

**计算公式**：
```
最终报价 = 剩余金额平均值 × 进度系数 × 随机浮动值
```

| 阶段 | 轮次 | 进度系数 |
|------|------|---------|
| 前期（压价） | 1-3 轮 | 0.6 ~ 0.7 |
| 中期（趋近） | 4-6 轮 | 0.8 ~ 0.9 |
| 后期（溢价） | 7 轮及以后 | 1.0 ~ 1.15 |

- **随机浮动**：±8%（小幅波动，避免报价机械重复）
- **兜底规则**：报价自动取整，小额取整至整数，大额以千为单位；报价永不为 0

## 3. 功能需求

### 3.1 游戏初始化
- 页面加载自动生成 26 个编号公文包（1-26 号）
- 随机分配 26 档奖金，每个箱子唯一不重复
- 「重新开始」按钮重置全部进度

### 3.2 选箱功能
- 开局强制选择 1 个专属箱子，选中后高亮锁定
- 专属箱子全程隐藏金额
- 未选箱时禁止开箱，有交互提示

### 3.3 开箱淘汰功能
- 点击未开启箱子，展示对应金额，标记为「已淘汰」
- 已开箱子置灰不可再次点击
- 奖金池实时更新：已开出金额划掉，剩余金额高亮
- 严格按轮次数量控制开箱次数，本轮开完自动触发银行家报价

### 3.4 银行家报价
- 每轮开箱结束，系统自动计算并弹出报价
- 采用 2.4 节标准化报价算法
- 弹窗两个按钮：**Deal**（成交）/ **No Deal**（不成交）
- 记录每轮报价历史，页面可查看

### 3.5 终极抉择
- 仅剩 2 箱时触发终极抉择弹窗
- 选择：保留当前箱子 / 交换箱子
- 选择后开箱结算，展示最终奖金

### 3.6 数据展示
- 实时显示：剩余奖金池、当前轮数、已开箱数量、历史报价
- 游戏结束展示：最高可中金额、最终获得金额、盈亏对比

## 4. UI 需求（综艺华丽风）

| 项目 | 内容 |
|------|------|
| **主色调** | 深海蓝底色 + 金色高光/文字，黑金蓝经典综艺配色 |
| **视觉特效** | 灯光渐变、卡片悬浮发光、选中高亮、开箱动态光影 |
| **主游戏区** | 26 个公文包卡片网格布局，未开/选中/已淘汰三种状态区分明显 |
| **奖金池** | 侧边栏展示 26 档金额，已淘汰灰色划掉，剩余金色高亮 |
| **状态提示区** | 金色字体展示当前轮次、本轮剩余开箱数、最新报价 |
| **决策弹窗** | 深色磨砂底 + 金色边框，贴合综艺风格 |
| **功能按钮** | 渐变金色样式，hover 有动效 |
| **适配** | PC 端为主（桌面端展示） |

## 5. 交互体验需求

- 开箱动画、报价弹窗弹出动画，还原综艺紧张氛围
- 操作限制：未完成本轮开箱无法触发下一步
- **新手引导**（轻量化，不做强制弹窗）：

| 阶段 | 提示文字 |
|------|---------|
| 开局 | "请选择你的专属箱子" |
| 选箱后 | "点击箱子完成本轮开箱" |
| 开箱完毕 | "等待银行家报价" |

## 6. 非功能需求

- 零后端、零数据库，纯前端本地运算，刷新即重置
- 代码轻量化，无冗余依赖，可拖拽部署 Netlify
- 逻辑严谨：金额不重复、轮次正常递进、报价算法稳定可复现

## 7. 版本规划

| 版本 | 内容 |
|------|------|
| **基础版（本次开发）** | 核心游戏逻辑 + 综艺风 UI + 基础交互 + 新手文字指引 + 标准化银行家报价算法 |
| **二期迭代（后续）** | 音效、多局累计统计、最高纪录、对局历史、暗黑模式/皮肤切换 |

`index.html` + `style.css` + `app.js` 完整源码，可直接本地打开 / 上传 Netlify 一键部署。
